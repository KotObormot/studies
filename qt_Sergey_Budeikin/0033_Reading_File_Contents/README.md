QIODevice — это базовый абстрактный класс во фреймворке Qt, который предоставляет единый интерфейс для всех устройств ввода-вывода, поддерживающих чтение и запись блоков данных.

Поскольку класс является абстрактным, вы не можете создать его экземпляр напрямую. Вместо этого вы всегда работаете с его конкретными наследниками. 

Главная цель QIODevice — абстрагировать программу от природы носителя информации (будь то файл на диске, сетевой сокет или массив байт в памяти), позволяя использовать для работы с ними одни и те же методы.

Основные наследники класса

Фреймворк Qt содержит множество готовых классов, которые наследуют функционал QIODevice:

->QFile — используется для работы с обычными файлами на диске.

->QTcpSocket / QUdpSocket — применяются для сетевого обмена данными по протоколам TCP и UDP.

->QBuffer — позволяет работать с обычным массивом байт (QByteArray) как со стандартным устройством ввода-вывода.

->QProcess — предназначен для запуска внешних программ и взаимодействия с их потоками ввода-вывода (stdin, stdout, stderr).

->QSerialPort — служит для обмена данными через последовательный интерфейс (COM-порт).


Ключевые методы

Любой класс, унаследованный от QIODevice, использует общий набор методов для управления жизненным циклом данных:

->open(OpenMode mode) — открывает устройство в определенном режиме (например, ReadOnly — только чтение, WriteOnly — только запись или ReadWrite).

->close() — закрывает устройство и освобождает ресурсы.

->read(char *data, qint64 maxSize) / readAll() — считывает указанное количество байт или все доступные данные целиком.

->write(const char *data) — записывает блок данных в устройство.

->seek(qint64 pos) — перемещает указатель позиции в устройстве (работает только для устройств с произвольным доступом, таких как файлы или буферы).


Два типа устройств в QIODevice

Класс разделяет все устройства ввода-вывода на две категории:

->Устройства с произвольным доступом (Random-Access Devices) — поддерживают поиск позиции (seek()). Вы можете перемещать курсор в любую точку (например, QFile, QBuffer).

->Последовательные устройства (Sequential Devices) — не поддерживают произвольный доступ. Данные считываются строго один раз по мере их поступления (например, сетевые сокеты QTcpSocket или каналы QProcess).


Важные сигналы

Так как QIODevice интегрирован в общую экосистему сигналов и слотов Qt, он предоставляет важнейшие асинхронные уведомления:
->readyRead() — срабатывает, когда в устройство поступает новая порция данных, готовая для чтения. Этот сигнал незаменим при работе с сетью, так как позволяет не блокировать основной поток программы в ожидании пакетов.

->bytesWritten(qint64 bytes) — генерируется каждый раз, когда порция данных успешно записывается в устройство.
